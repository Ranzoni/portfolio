Case profissional

Bertoli HUB - Produto Digital de Curso de Turismo Privativo

Plataforma online proprietária da Bertoli Tours para centralizar e distribuir conteúdos educacionais com autonomia total de gestão, controle de acesso e evolução contínua do produto.

Voltar para portfólio Falar sobre um projeto

O Bertoli HUB surgiu da necessidade de oferecer uma solução própria, sem dependência de plataformas externas, garantindo controle total sobre usuários, conteúdos e regras de acesso.

Desenvolvi a aplicação de ponta a ponta como fullstack, sendo responsável pela arquitetura do sistema, implementação das funcionalidades e configuração da infraestrutura de deploy.

A plataforma permite autenticação de usuários e controle de acesso baseado em planos (HUB com acesso geral ou por módulos específicos), além de oferecer uma experiência contínua de aprendizado com dashboard personalizado, retomada de conteúdo e integração com player de vídeo via Vimeo.

Também foi implementada uma área administrativa exclusiva, permitindo ao cliente gerenciar toda a plataforma de forma autônoma, incluindo cursos, módulos, aulas, usuários, professores, notícias e ajustes básicos de layout.

Arquitetura e tecnologias

  • Back-end em .NET (API REST), containerizado com Docker.
  • Banco de dados PostgreSQL com migrações via Entity Framework.
  • Front-end em React.js (HTML e CSS).
  • Infraestrutura em servidor Ubuntu (Droplet).
  • Nginx como servidor web e proxy reverso.
  • Deploy automatizado com scripts shell (.sh).

Funcionalidades

  • Sistema de autenticação e gerenciamento de usuários.
  • Controle de acesso por planos e módulos.
  • Dashboard com progresso e continuidade de aulas.
  • Integração com player de vídeo (Vimeo via iframe).
  • Modo claro e escuro.
  • Sistema de notícias.
  • Área administrativa completa (CMS próprio).

Decisões técnicas

A arquitetura foi planejada para garantir escalabilidade e manutenção simplificada: utilização de API REST em .NET para estrutura robusta e performática, containerização com Docker para padronização de ambiente, separação entre front-end e back-end para evolução independente, Nginx como proxy reverso para melhor gerenciamento de requisições e deploy automatizado para mais agilidade e confiabilidade nas atualizações.

Resultado

O projeto foi entregue como uma solução completa e funcional, permitindo ao cliente total autonomia na gestão de conteúdos e usuários, sem dependência técnica para operação da plataforma.
